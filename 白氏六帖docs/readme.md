白孔六帖_结构化.json 说明文档
1. 文件简介
白孔六帖_结构化.json 是《白孔六帖》资料的结构化整理结果。
文件顶层为 JSON 数组，数组中的每个对象代表一条独立词条记录，适合用于检索、统计分析、部类研究，以及与其他古籍或诗文语料进行对照。

2. 数据概况
文件名：白孔六帖_结构化.json
编码：UTF-8
顶层结构：List[Object]
记录总数：55,823
唯一卷名数：96
唯一部类数：1,370
作者字段取值：白 共 55,190 条，孔 共 633 条
部类附名 非空记录：18,463 条
注文 非空记录：50,297 条
说明：

卷名范围覆盖卷一至卷一百，但当前数据中缺少卷五、卷二十四、卷八十一、卷八十二。
作者 字段目前仅见 白 与 孔 两种取值，可作为区分不同来源层次的重要标记；如需进一步学术解释，建议结合原始整理规则再作界定。
3. 字段结构
每条记录包含以下字段：

字段名	类型	含义	示例
卷	string	卷次，使用中文数字表示	一
部类	string	词条所属主部类	天
部类序号	string	该部类在当前卷中的序号，使用中文数字表示	一
部类附名	string | null	对部类的补充说明、附属名目或细分标签	地理土附
作者	string	当前记录的来源标记，目前见 白、孔	白
词条	string	结构化后的核心词条或摘句	天尊
注文	string | null	与词条对应的注文、释文或续句	地卑

5. 使用建议
读取时请显式指定 UTF-8 编码，避免中文或异体字显示异常。
卷 与 部类序号 均使用中文数字；若要进行排序、分卷统计或可视化，建议先转换为阿拉伯数字字段。
部类附名 与 注文 存在 null 或空值，处理时应做好缺失值判断。
卷号并非连续齐备，不能默认按 1 到 100 完整展开。
文本中保留了部分古籍常见字形或异体字，如 髙、隂、黙 等；若要进行现代汉字归一化，建议保留原文字段，另建标准化字段，避免覆盖原始信息。
6. Python 读取示例
import json
from pathlib import Path

path = Path("白孔六帖_结构化.json")

with path.open("r", encoding="utf-8") as f:
    data = json.load(f)

print(f"记录总数：{len(data)}")
print(data[0])
